# Block 04 — Internationalization

Implements the i18n module: locale file validation, compile-time `t()` key checking, interpolation variable verification, and codegen that wires locale bundles into the generated Nuxt app.

Part of M3. Opt-in: activates when `"i18n"` is listed in `modules` in `origami.toml`. When inactive, `t()` is a compile error.

## Error codes

| Code | Condition | Phase |
|------|-----------|-------|
| CLT401 | `t()` key not found in default locale file | analyzer |
| CLT402 | Interpolation variable missing from `t()` call | analyzer |
| CLT403 | Locale key exceeds 3 segments | i18n validation |
| W101 | Key missing from a non-default locale file | i18n validation |
| W102 | Extra key in a non-default locale file | i18n validation |
| W103 | Extra interpolation variable passed to `t()` | analyzer |

CLT401 and CLT402 are errors — unknown keys and missing variables are always bugs.
W101 and W102 are warnings — missing translations are a product decision, not a compiler error.

## Locale files

Live in `locales/` at workspace root, shared across all apps.

```
locales/
  ├── en.json     ← default locale, canonical key source
  └── it.json
```

### Key naming convention

```
{context}.{component}.{label}
```

- `context`: page name or `common` for shared strings (`books`, `admin`, `common`)
- `component`: component or feature name, camelCase (`bookCard`, `filterForm`)
- `label`: string identifier (`title`, `saveButton`, `emptyState`)

```json
{
  "common.loading":           "Loading...",
  "common.save":              "Save",
  "books.bookCard.title":     "Book",
  "books.bookCard.author":    "by {author}",
  "books.list.count":         "{count, plural, one {# book} other {# books}}"
}
```

Keys are flat strings. The 3-segment limit is enforced at the key string level — no JSON nesting.
A key like `"admin.users.table.header.name"` (5 segments) is CLT403 regardless of position.

Only `one` / `other` plural forms are supported. Complex ICU plural rules (`few`, `many`, `zero`) are deferred.

### Interpolation

`{variableName}` for substitution. `{variableName, plural, one {…} other {…}}` for plurals.

### Cross-locale validation

The default locale (`en.json`) is the only source of truth for which keys exist.

| Condition | Code | Severity |
|-----------|------|----------|
| Key in default, missing from other locale | W101 | Warning |
| Key in other locale, not in default | W102 | Warning |
| Key used in source but not in default locale | CLT401 | Error |

## Compiler pipeline extensions

### Lexer
No changes. `t()` uses the same token stream as any function call.

### Parser — TCall node

`t()` is a built-in recognised by name. Stored as a `TCall` node:

```
TCall
  ├── key: String            ← e.g. "books.list.title"
  └── args: Option<TArgs>    ← interpolation variables
```

Valid in template prop values and logic section expressions.
If the i18n module is not enabled, any `t()` call is a parse error.

### Analyzer — compile-time validation

Validates every `TCall` against the `LocaleManifest`:

- CLT401: key not in default locale
- CLT402: required interpolation variable missing from `t()` call
- W103: extra variable passed to `t()` with no matching placeholder

## origami-i18n crate

New crate. Input: locale files + `origami.toml`. Output: `LocaleManifest`.

### Input

```
locales/
  ├── en.json         ← parsed as default locale
  └── *.json          ← all others validated against default

origami.toml
  └── compiler.default_locale: String   ← "en"
```

### Output

```rust
LocaleManifest {
    default_locale: String,
    keys: HashMap<String, LocaleKey {
        value: String,
        placeholders: Vec<String>,   // extracted at parse time, e.g. "by {author}" → ["author"]
    }>,
    locales: Vec<LocaleFile>,        // all non-default locales
}
```

`placeholders` are extracted from key values at parse time. The analyzer uses this to validate `t()` calls without re-parsing locale files.

## origami-codegen extension

Receives `LocaleManifest`, emits:

```
__generated__/
  └── locales/
      ├── en.json
      └── it.json
```

Extends generated `nuxt.config.ts`:

```typescript
export default defineNuxtConfig({
  modules: ['@nuxtjs/i18n'],
  i18n: {
    locales: ['en', 'it'],
    defaultLocale: 'en',
    langDir: './locales/'
  }
})
```

`t()` calls in generated `.vue` files use `useI18n()` destructured in `<script setup>`:

```typescript
// always emitted when i18n is active
const { t } = useI18n()
```

RULE: Never emit `$t()` — it is a legacy global that bypasses the Composition API.

## CLI contribution

`origami check`: validates locale file JSON, segment count (CLT403), cross-locale diff (W101, W102).

`origami build`: extended to invoke `origami-i18n` and write `__generated__/locales/`.

`origami dev` and `origami build` gain the `--locale <locale|all>` flag:
- Default: `all` (all locales loaded, runtime switching available)
- `--locale it`: loads Italian only (faster startup / smaller bundle)
